# v0.2 Agent Event Protocol

## Goal

Introduce a unified event protocol for Browser Agent Runtime.

The current MVP focuses on executing a single action plan. v0.2 introduces an event-driven runtime where the UI, provider, and executor share the same agent lifecycle.

## Agent lifecycle

```text
Task
 |
 v
Observe
 |
 v
Plan
 |
 v
Action
 |
 v
Verify
 |
 v
Complete / Retry
```

## Core event model

```ts
export type AgentEvent =
  | ObserveEvent
  | ThinkingEvent
  | PlanEvent
  | ActionEvent
  | VerifyEvent
  | ErrorEvent;
```

## Event examples

### Observe

```json
{
  "type": "observe",
  "snapshotId": "snapshot-001",
  "message": "Analyzing current page"
}
```

### Action

```json
{
  "type": "action",
  "action": "fill",
  "targetRef": "element-3",
  "status": "running"
}
```

### Verify

```json
{
  "type": "verify",
  "success": true,
  "changes": [
    "input value updated"
  ]
}
```

## UI consumers

The side panel consumes events to render:

- Agent timeline
- Streaming status
- Current action
- Verification result

Example:

```text
Agent

✓ Observe page
✓ Find input
⚡ Filling content
○ Verify result
```

## Runtime boundaries

```text
Chrome Extension
        |
        v
Agent Event Stream
        |
 +------+------+
 |             |
 UI        Executor
 |             |
Timeline    DOM Action
```

## Implementation order

1. Add event types shared by bridge and extension.
2. Convert provider responses into AgentEvents.
3. Render event timeline in Side Panel.
4. Add observe-act loop consuming events.
5. Add snapshot diff verification.

## Non-goals

- No unrestricted JavaScript execution.
- No direct model-controlled selectors.
- No hidden browser actions without approval.
