# v0.2 Agent Event Protocol

## Goal

Introduce a unified event protocol for Browser Agent Runtime.

The current MVP focuses on executing a single action plan. v0.2 introduces an event-driven runtime where the UI, provider, and executor share the same agent lifecycle.

## Runtime lifecycle

```text
Task
 |
 v
Action
 |
 v
Verify
 |
 v
Complete / Retry
```

Observe and plan still happen internally, but they are not emitted as static UI events. The side panel should show only information that changes or proves progress.

## Core event model

```ts
export type AgentEvent =
  | ThinkingEvent
  | ActionEvent
  | VerifyEvent
  | CompleteEvent
  | ErrorEvent;
```

## Event examples

### Action

```json
{
  "type": "action",
  "action": "fill",
  "targetRef": "element-3",
  "status": "running"
}
```

### Verify

```json
{
  "type": "verify",
  "success": true,
  "changes": [
    "input value updated"
  ]
}
```

## UI consumers

The side panel consumes events to render:

- meaningful streamed model output
- Current action
- Verification result
- Completion or error state

Example:

```text
Agent

⚡ Filling content
✓ Input value updated
✓ Task complete
```

## Runtime boundaries

```text
Chrome Extension
        |
        v
Agent Event Stream
        |
 +------+------+
 |             |
 UI        Executor
 |             |
Timeline    DOM Action
```

## Implementation order

1. Add event types shared by bridge and extension.
2. Stream provider output and runtime results as AgentEvents.
3. Render event timeline in Side Panel.
4. Add the internal observe-act loop.
5. Add snapshot diff verification.

## Non-goals

- No unrestricted JavaScript execution.
- No direct model-controlled selectors.
- No hidden browser actions without approval.
